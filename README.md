# Human vs Random Sequence Detector

An interactive machine learning application that determines whether a binary sequence was generated by a human or is truly random.

Humans are surprisingly bad at producing randomness — this project models that behaviour using statistical features and probabilistic learning.

## Overview

This project combines feature engineering and probabilistic modelling to identify patterns in sequences that appear random but contain subtle human biases.

The model analyses properties such as entropy, transition behaviour, and structural patterns, then classifies sequences using a Gaussian Naive Bayes classifier.

Users can interact with the system through a web interface and attempt to "beat the model" by generating sequences that appear random.

For the project write-up, evaluation results, and limitations, see [REPORT.md](REPORT.md).

---

## Key Features

- Entropy and randomness analysis  
- Markov transition modelling  
- KL divergence for distribution comparison  
- Run-length and alternation pattern detection  
- Bayesian classification (Gaussian Naive Bayes)  
- Multi-sequence prediction using log-likelihood aggregation  
- Interactive web app with gamified interface  
- User data logging for future model improvement  

---

## How It Works

1. Synthetic data is generated to simulate common human behaviours such as alternation, repetition, and bias  
2. Each sequence is converted into statistical features  
3. A probabilistic model classifies sequences as human or random  
4. Predictions across multiple sequences are combined to improve accuracy

## Key Learnings

- Feature engineering is critical for distinguishing structured vs random data  
- Probabilistic models provide stable and interpretable predictions  
- Aggregating multiple inputs significantly improves robustness  
- More complexity does not always improve model performance
